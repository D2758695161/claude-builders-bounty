# claude-review — AI-Powered PR Reviewer

A CLI tool + GitHub Action that uses Claude to analyze GitHub pull requests and produce structured code reviews.

## Features

- **CLI**: `claude-review --pr <url>` — one-command PR analysis
- **GitHub Action**: automate reviews on every PR or trigger manually
- **Structured output**: Summary → Risks → Suggestions → Confidence Score
- **Posts directly as PR comment** with `--post-comment`

## CLI Setup (2 steps)

```bash
# 1. Install
pip install requests   # or: curl -o claude-review.py <url>

# 2. Run
export GITHUB_TOKEN=ghp_your_github_token
export ANTHROPIC_API_KEY=sk-ant-your_anthropic_key
python claude-review.py --pr https://github.com/owner/repo/pull/123
```

### CLI Options

| Flag | Description |
|------|-------------|
| `--pr <url>` | GitHub PR URL (required) |
| `--github-token <token>` | GitHub PAT (or `GITHUB_TOKEN` env) |
| `--anthropic-key <key>` | Anthropic API key (or `ANTHROPIC_API_KEY` env) |
| `--post-comment` | Post review as a PR comment |
| `-o <file>` | Write review to file instead of stdout |

## GitHub Action Setup (5 steps)

```yaml
# .github/workflows/pr-review.yml
name: AI PR Reviewer

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Install
        run: pip install requests

      - name: Review PR
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          python claude-review.py \
            --pr "${{ github.event.pull_request.html_url }}" \
            --post-comment
```

**Required secrets**: `ANTHROPIC_API_KEY` (get from [Anthropic console](https://console.anthropic.com/))

```markdown
**Repository**: owner/repo  
**Author**: @developer  
**Files changed**: 5 (+120 / -45)  
**Review generated by**: Claude (claude-sonnet-4-20250514) via `claude-review` CLI

---

### 📋 Summary
This PR fixes a session token expiration bug in the authentication middleware...

### ⚠️ Risks
- No test coverage added for the new token refresh logic
- Could introduce race condition if two concurrent requests both trigger refresh

### 💡 Improvement Suggestions
- Add unit test for `TokenRefreshMiddleware`
- Consider adding a lock to prevent concurrent refresh attempts

### 🎯 Confidence Score
**Medium** — The fix looks sound but lacks test coverage

### 📝 Additional Notes
Would benefit from an integration test covering the full auth flow.
```

## How It Works

1. Fetches PR metadata + diff via GitHub REST API
2. Sends diff + context to `claude-sonnet-4-20250514` via Anthropic API
3. Returns structured Markdown review
4. Optionally posts as PR comment
