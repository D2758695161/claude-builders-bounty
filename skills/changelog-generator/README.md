# CHANGELOG Generator Skill

A pure-bash script that generates a structured `CHANGELOG.md` from git commit history.

## Acceptance Criteria

✅ Works via `bash changelog.sh`  
✅ Fetches commits since last git tag  
✅ Auto-categorizes into: Added / Fixed / Improved / Changed / Removed  
✅ Outputs properly formatted `CHANGELOG.md`  
✅ Conventional commits support  
✅ README with 3-step setup  

## Usage

```bash
# Auto-detect repo (current directory)
bash changelog.sh

# Specify repo path and output file
bash changelog.sh /path/to/repo CHANGELOG.md
```

## Sample Output

```markdown
# Changelog

*Auto-generated from git history — since v1.2.0*

Generated: 2026-04-13

## Added
- User authentication feature *([`a1b2c3d`])*
- Password reset flow *([`b2c3d4e`])*

## Fixed
- Login redirect bug *([`c3d4e5f`])*

## Changed
- Updated dependencies *([`d4e5f6g`])*

---
*Generated by changelog.sh — Claude Code skill*
```

## How It Works

1. Detects the last git tag → uses commits since that tag
2. Falls back to last 50 commits if no tags
3. Parses conventional commit format: `type(scope): message`
4. Categorizes by type prefix (feat→Added, fix→Fixed, etc.)
5. Renders Markdown with commit hashes and author attribution

## Install

```bash
# Step 1: Copy the script
cp changelog.sh /usr/local/bin/changelog.sh

# Step 2: Make it executable  
chmod +x /usr/local/bin/changelog.sh

# Step 3: Run from any repo
cd /path/to/your/repo && changelog.sh
```
